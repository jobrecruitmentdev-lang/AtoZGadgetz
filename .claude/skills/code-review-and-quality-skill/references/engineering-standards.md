# Engineering Standards

## DRY Principles (Don't Repeat Yourself)
* Abstract shared logic into reusable modules, utilities, or custom hooks.
* If a block of code is duplicated three times, it MUST be extracted into a common function.

## Clean Code Rules
* Write code for humans first, machines second. Prioritize readability.
* Prefer explicit, easy-to-understand logic over clever, implicit side-effects.
* Keep cyclomatic complexity low (avoid deep nesting of if/else statements).

## Patterns
* Use Dependency Injection where possible.
* Rely on established structural patterns rather than ad-hoc solutions.
